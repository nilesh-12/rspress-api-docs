Authentication#

Prerequisites:

  • You must need a client token to access the API. Which you can obtain from out admin.

  • You must create a user and obtain an API key

API Keys

Authentication is a critical aspect of using the API, ensuring that only authorized users can access certain functionalities. The primary method of authentication is through API keys.

How API Keys Work

  1. Generation: Users can generate API keys from their account dashboard.
  2. Usage: Include the API key in the header of each request as follows: X-Api-Key: YOUR_API_KEY
  3. Security: Keep your API key confidential. Do not expose it in client-side code or public repositories.

Client Token

  1. Generation: Users can get Client Token from our admin.

  2. Usage: Include the API key in the header of each request as follows: X-Client-Token: YOUR_CLIENT_TOKEN

  3. Security: Keep your Client Token confidential. Do not expose it in client-side code or public repositories.

Errors#

The API uses standard HTTP status codes to indicate the result of API requests.

Status Code Meaning
200 OK - The request was successful.
400 Bad Request - The request was invalid.
401 Unauthorized - Authentication failed.
403 Forbidden - Access is denied.
404 Not Found - The resource does not exist.
422 Unprocessible Entity - Data validation failed.
500 Internal Server Error - A server error occurred.

Error Object When an error occurs, the API returns an error object in the response body, which includes the following fields:

  • code: A numeric code representing the error type.
  • message: A human-readable description of the error.
  • details: Additional information about the error (if available).

Create a template#

POST /upload_template

Creates a new template file or replaces an existing one.

Parameters:

templateNamestringRequired

The name of the template.

templateFileHTML fileRequired

The file to upload.

filetypeJSON fileRequired

The schema to validate the data.

POST /upload_template
# using cURL
curl -X POST https://nduepptfsgpmdepcwna5tatdou.apigateway.ap-mumbai-1.oci.customer-oci.com/pdf-gen/v1/upload_template \
  -H "X-Api-Key: YOUR_API_KEY" \
  -H "X-Client-Token: YOUR_CLIENT_TOKEN" \
  -H "Content-Type: multipart/form-data" \
  -F "templateName=main_v1" \
  -F "templateFile=@main_v1.html" \
  -F "validationSchema=@main_v1.schema.json"
200 OK
{
  "message": "Template uploaded successfully"
}

List all templates#

Retrieves a list of templates.

Parameters:

  -H "X-Client-Token: YOUR_CLIENT_TOKEN" \

{
